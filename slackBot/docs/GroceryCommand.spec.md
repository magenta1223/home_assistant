# GroceryCommand 기능 명세

## 목적

구매 패턴을 분석하여 식재료 부족 시점을 예측하고 가족 구성원에게 알림을 발송한다.

- **기록 대상**: 구매 날짜와 수량만 기록
- **예측 방식**: 구매 이력의 평균 소비 주기를 계산하여 다음 구매 시점 예측
- **알림 조건**: 예측 잔여일이 임박하거나 초과했을 때 Slack DM 발송 (스케줄러 별도 구현)

실시간 사용량 추적(매 사용 시 입력)은 현실적으로 불가능하므로 사용하지 않는다.

---

## DB 스키마

### `grocery_items`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INTEGER PK | 자동 증가 |
| `name` | TEXT UNIQUE | 식재료 이름 |
| `unit` | TEXT | 단위 (기본값: `개`) |

### `grocery_purchases`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INTEGER PK | 자동 증가 |
| `item_id` | INTEGER FK | `grocery_items.id` 참조 |
| `qty` | REAL | 구매 수량 (항상 양수) |
| `purchased_at` | TEXT | 구매 일시 (기본값: 현재 시각, localtime) |

---

## 예측 알고리즘

구매 이력이 **2회 이상**인 경우에만 예측을 수행한다.

1. 아이템별 구매 이력 조회 (날짜 오름차순)
2. 연속 구매 간격의 평균 계산 (`avg_interval_days`)
   - 예: [1월 1일, 1월 15일, 2월 5일] → 간격 [14일, 21일] → 평균 17.5일
3. 마지막 구매일로부터 경과 일수 계산 (`days_since_last`)
4. 잔여 예측일 계산: `days_remaining = avg_interval_days - days_since_last`
5. 상태 판정:
   - `days_remaining <= 0` → ⚠️ 부족 예상 (이미 구매 주기 초과)
   - `days_remaining <= 3` → 🔔 구매 임박
   - `days_remaining > 3` → ✅ 여유 있음

구매 이력이 **1회 이하**이면 "데이터 부족" 메시지를 표시한다.

## 명령어 명세

### `/구매 <이름> <수량><단위>`

구매한 식재료를 기록한다.

- **형식**: `/구매 달걀 10개`
- **지원 단위**: `개`, `L`, `리터`, `g`, `kg`, `봉`, `팩`, `병`, `캔`, `줄`, `판`, `묶음`, `포`
- **동작**:
  1. `grocery_items`에 아이템이 없으면 자동 생성
  2. `grocery_purchases`에 구매 기록 삽입
- **응답**: `in_channel` — 가족 모두에게 구매 사실 공유

### `/재고`

모든 식재료의 예측 현황을 표시한다.

- **동작**: 각 아이템의 구매 이력을 분석하여 잔여 예측일 계산
- **응답**: `ephemeral` — 조회한 사람에게만 표시

**응답 예시**:
```
재고 현황

⚠️ 달걀: 마지막 구매 2일 전, 평균 14일 주기 (약 -2일 남음)
🔔 우유: 마지막 구매 12일 전, 평균 14일 주기 (약 2일 남음)
✅ 쌀: 마지막 구매 5일 전, 평균 30일 주기 (약 25일 남음)
📊 소금: 구매 이력 부족 (1회) — 예측 불가
```

## 알림 정책 (향후 구현)

> 이번 스코프 밖. 예측 로직은 `/재고` 커맨드에 포함하고, 스케줄러는 별도 태스크로 구현 예정.

- **도구**: `node-cron` 등 스케줄러
- **주기**: 매일 오전 9시 실행
- **조건**: `days_remaining <= 3`인 아이템이 있을 때 가족 채널에 DM 발송
- **내용**: 부족 예상 식재료 목록 + 예측 잔여일

## 검증 방법

1. `npm run typecheck` — 타입 에러 없음
2. `npm test` — 전체 테스트 통과
3. `/구매 달걀 10개` → `grocery_purchases` 테이블에 행 생성 확인
4. 구매 2회 이상 후 `/재고` → 예측 잔여일 표시 확인
5. 구매 1회 후 `/재고` → "데이터 부족" 메시지 표시 확인
